# View

<Meta title="Core/Components/Views/View" />
export const Template = args => <View {...args} />

`View` is an organizational component that manages chart state. It does not display any visual elements on a chart, but it prepares data for display. In particular, it defines scales for the axes and prepares data for plotting.

<ArgsTable of={View} />

## Coordinate systems

To see the effect of `View`, consider an `Axis` component drawn by itself (below, first example) and within a `View` component (below, second example).

<Canvas>
    <Story name="without view">
        <Chart
            size={[400, 300]}
            padding={[40, 40, 40, 40]}
            style={{ margin: '0.5em', border: 'solid 1px #aa3333', display: 'inline-block' }}

    <Story name="with view">
        <Chart
            size={[400, 300]}
            padding={[40, 40, 40, 40]}
            style={{ margin: '0.5em', border: 'solid 1px #aa3333', display: 'inline-block' }}
        >
            <Surface variant={'inner'} />
            <View container={{ padding: [20, 20, 0, 0] }}>
                <Axis variant={'bottom'} />
            </View>
        </Chart>
    </Story>
</Canvas>

In the first example above, without `View`, the axis does not have information about the scale or the extent of the axis. It draws some ticks and labels, but the result is unsatisfactory.

In the second example above, a `View` component specified the extent of the drawing area and the scale for the axes. The `Axis` component inside `View` can thus produce informative elements.
